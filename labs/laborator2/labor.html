Tag-urile de tip form

Are rolul de a contine informatii introduse de utilizator. Acestea mai departe vor fi trimise la server. (vom vedea in laboratoarele viitoare si cum)

Contine taguri de tip input. Tagurile de tip input primesc diferite tipuri de informatii: email, text, parola.

In functie de tipul de informatie inputul se va comporta diferit. De exemplu, tag-ul de tip password o sa inlocuiasca, in vizual, caracterele cu "*", tagurile de tip date vor deschide un calendar pentru a introduce odata, iar cele de tip email nu vor permite decat introducerea unui email.

Probabil in laboratorul anterior ati vazut ca fiecare tag poate avea propietati/atribute, de exemplu: id, o propietate unica care identifica tagul respectiv

<h1 id="id_titlu"></h1>

Un tag de tip input are urmatoarele propietati caracteristice (mai importante):

  • name: identificator unic la nivel de form. Dupa acest atribut, serverul care primeste form-ul o sa identifice valoarea
  • value: daca vrei sa definesti o valoare din default
  • placeholder: o indicatie in background care poate da un indiciu utilizatorului despre ce ar trebuii sa scrie inauntru

Cateva exemple:

<form> <h2>Login</h2> <label>Username: </label><input name="username" type="text" placeholder="your username here" /> <label>Password: </label><input name="password" type="password" placeholder="your password" /> <input type="submit" /> // apasarea butonului submit </form> O sa arate ceva in genul:

CSS

Este un limbaj folosit pentru stilizarea HTML-ului

Putem schimba culori, dimensiuni, sa ascundem elementele sau chiar sa facem un design diferit pentru telefon, tableta, laptop, PC etc. (in functie de dimensiunea ecranului)

Se poate insera in 3 feluri intr-un fisier html

  • Prin inline style pe fiecare element in parte

    • Stilul inline are ca dezavantaj faptul ca este greu de urmarit. Daca avem fisiere html cu sute de linii, o sa ne fie foarte greu “sa mentinem codul pe termen lung”. De asemenea, daca vrem de exemplu ca toate paragrafele sa fie scrie cu bold, trebuie sa aplicam aceeasi regula pe fiecare paragraph ( cod duplicat). Cand ne hotaram ca vrem ca paragrafele sa fie cu italic in loc de bold, trebuie sa modificam iar in sute de locuri. OBS2: La proiect nu veti avea voie sa folositi stil inline

    <h1 style="color: red;"></h1> <!-- Daca vrem sa aplicam 2 reguli diferite pentru acelasi element, le separam prin ; -->
  • Prin tagul style

    • Este definit in HEAD, in interiorul unui tag style.

    Avantajul fata de CSS-ul inline este ca daca vrem ca toate paragrafele sa fie de culoare rosie si cu font de 25px scriem regula o singura data, nu pentru fiecare paragraf in parte. Daca vrem sa schimbam culoare din rosu in verde, atunci modificam intr-un singur loc.

    <head> <style> /* toate tagurile de tip h1 vor avea propietatiile de css */ h1 { color: red; font-size: 25px; } /* toate tagurile care au clasa "titlu" setata prin atribut vor avea propietatiile de css */ .titlu { color: red; font-size: 25px; } /* toate tagurile care au clasa titlu setata prin atribut vor avea propietatiile de css */ #titlu_principal { color: red; font-size: 25px; } </style> </head>
  • Prin fisier extern:
    • <head> <link rel="stylesheet" type="text/css" href="mystyle.css"> </head>

    unde mystyle.css se afla in acelas loc cu fisierul tau html

    Metoda de la style tag are ca dezavantaj faptul ca face fisierul html si mai lung. Daca avem 500 de linii de HTML si mai adaugam inca 600 de linii de CSS o sa avem in final un fisier foarte mare. Mai mult, daca vrem sa aplicam acelasi stil pentru fisierele index.html si home.html, trebuie sa-i dam COPY PASTE in fiecare fisier. De acea avem nevoie de fisier extern

    Trebuie sa:

    • Intram in Notepad++/ VSCode si sa creem un fisier index.html
    • Facem un fisier index.css (aveti grija sa nu-l salvati ca index.css.txt)
    • Mergem in index.html si adaugam in head o legatura catre fisierul nostru CSS

    Obs: rel si type raman mereu la fel. Se modifica doar atributul href cu “calea” catre fisierul nostru css.

    Obs: Dupa legarea fisierului extern de css inchideti si redeschideti browserul
